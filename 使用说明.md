# 待办事项 — 使用说明

## 快速开始

### 启动应用

```bash
cd 项目目录
python3 server.py
```

浏览器会自动打开 `http://localhost:8765/index.html`。

如果服务器已在运行，再次执行上述命令会自动打开浏览器，不会重复启动。

### 停止应用

关闭浏览器标签页即可。服务器进程会在后台保持运行，下次打开浏览器访问 `http://localhost:8765/index.html` 即可继续使用。

如需彻底停止服务器：
```bash
lsof -ti tcp:8765 | xargs kill
```

## 基本操作

### 添加待办事项
点击 **「+ 添加待办」** 按钮，填写内容、截止日期（可选）、优先级（1-3 星），点击「确认添加」。

### 编辑待办事项
点击事项右侧的 ✏️ 按钮，修改后点击「保存」。

### 完成待办事项
点击事项右侧的 ✅ 按钮，事项会移入「已完成」列表。

### 删除待办事项
点击事项右侧的 🗑 按钮，确认后删除。

### 排序
在待办列表顶部可选择排序方式：
- **添加时间**（默认）
- **截止日期**
- **优先级**

点击排序方向箭头 ↑↓ 切换升序/降序。

### 备注
已完成事项可以添加备注：点击「+ 备注」，输入内容后按回车保存。

### 导出 / 导入数据
- **📥 下载备份**：将数据导出为 JSON 文件
- **📂 导入 JSON**：从之前导出的 JSON 文件恢复数据（需要 Chrome/Edge 浏览器）
- **💾 另存为**：导出到指定位置

## 数据存储

### 数据文件位置

所有数据存储在项目目录下的独立 JSON 文件中：

```
项目目录/dist/To Do.app/Contents/Resources/
├── index.html      ← 界面模板
├── todos.json      ← 待办事项数据
├── server.log      ← 服务器日志
└── backups/        ← 自动备份目录
    ├── todos_20260610_085454.json
    ├── todos_20260610_090102.json
    └── ...
```

> **重要**：`todos.json` 是你所有待办事项的存储文件。建议定期备份此文件。

---

## 数据保护机制

本应用内置了多层数据保护：

### 1. 自动备份
每次保存数据时，系统会自动在 `backups/` 目录下创建一份带时间戳的备份文件，保留最近 **20 份** 备份。

```
触发时机：
- 每次添加、编辑、完成、删除事项
- 应用启动时
```

### 2. 原子写入
数据先写入临时文件（`.tmp`），写入成功后才替换原文件。即使在保存过程中断电或崩溃，`todos.json` 也不会损坏。

### 3. 空数据保护
如果浏览器意外发送了空数据（例如旧标签页），服务器会拒绝保存，数据不会丢失。

### 4. 数据模板分离
界面模板（`index.html`）和数据文件（`todos.json`）完全独立。更新软件界面不会影响你的数据。

## 数据恢复

### 从自动备份恢复

如果发现数据丢失或异常，按以下步骤恢复：

**步骤 1：** 打开备份目录
```
项目目录/dist/To Do.app/Contents/Resources/backups/
```

**步骤 2：** 找到最新的备份文件（按时间排序，最新的在最后）：
```
todos_20260610_090102.json   ← 最新
todos_20260610_085930.json
todos_20260610_085454.json   ← 最早
```

**步骤 3：** 将备份文件复制覆盖 `todos.json`：
```bash
cp backups/todos_20260610_090102.json ../todos.json
```

**步骤 4：** 刷新浏览器页面即可看到恢复的数据。

### 从手动备份恢复

如果你之前使用「下载备份」功能导出了 JSON 文件：

1. 打开浏览器，点击「📂 导入 JSON」按钮
2. 选择你之前导出的备份文件
3. 页面会自动加载备份数据

### 数据丢失的常见原因及处理

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 所有事项消失 | 浏览器旧标签页覆盖 | 从 `backups/` 恢复最新备份 |
| 部分事项丢失 | 保存时网络中断 | 从 `backups/` 恢复 |
| 软件更新后数据没了 | 更新过程误覆盖 | `todos.json` 不受影响，如丢失从 `backups/` 恢复 |
| 文件被误删 | 手动删除或清理工具 | 从 `backups/` 或手动备份恢复 |

## 注意事项

1. **定期手动备份**：定期将 `todos.json` 复制到其他位置（如 iCloud、U 盘）
2. **不要手动编辑 `todos.json`**：错误格式可能导致数据读取失败
3. **迁移项目时**：整个 `项目目录/dist/To Do.app/Contents/Resources/` 目录一起复制，确保数据和备份都迁移
4. **浏览器兼容性**：推荐使用 Chrome 或 Safari。导入/导出功能需要 Chrome/Edge（Safari 不支持 File System Access API）
