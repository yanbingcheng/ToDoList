# 待办事项 — 开发文档

## 项目概述

一个基于 Python HTTP 服务器 + 浏览器前端的本地待办事项管理工具，支持打包为 macOS `.app` 应用。

- **技术栈**：Python 3 + HTML/CSS/JavaScript（原生，无框架依赖）
- **打包工具**：py2app
- **端口**：8765

---

## 架构设计

### 数据流

```
┌─────────┐     HTTP      ┌──────────┐    文件读写    ┌────────────┐
│ 浏览器   │ ←───────────→ │ server.py │ ←───────────→ │ todos.json  │
│ (Safari/ │  GET /api/    │ (Python)  │   JSON        │ (数据文件)   │
│  Chrome) │  POST /api/   │           │               ├────────────┤
│          │               │           │               │ backups/    │
│          │  GET /index   │           │               │ (自动备份)   │
│          │               │           │               └────────────┘
└─────────┘               └──────────┘
```

### 服务器 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/index.html` | GET | 返回纯模板 HTML（无缓存） |
| `/api/todos` | GET | 返回 todo 列表 JSON |
| `/api/todos` | POST | 保存 todo 列表 JSON（空数组时检查保护） |
| `/ping` | GET | 健康检查 |

### 两种运行模式

| 模式 | 启动方式 | DATA_DIR |
|------|---------|----------|
| **源码模式** | `python3 server.py` | `dist/To Do.app/Contents/Resources/` |
| **打包模式** | 双击 `To Do.app` | `To Do.app/Contents/Resources/`（内嵌） |

源码模式下服务器以后台守护进程方式运行（`--serve` 参数），主进程启动守护进程后打开浏览器。

## 数据库设计

### 数据文件：`todos.json`

```json
[
  {
    "id": "mq5yjbt91ii14s",
    "content": "待办事项内容",
    "deadline": "2026-06-19",
    "priority": 3,
    "createdAt": "2026-06-09T01:24:50.445Z",
    "completedAt": null,
    "notes": "",
    "notesEditing": false
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识符（`Date.now().toString(36) + random`） |
| `content` | string | 事项内容 |
| `deadline` | string | 截止日期（`YYYY-MM-DD`，可选） |
| `priority` | int | 优先级（1-3，3 最高） |
| `createdAt` | ISO string | 创建时间 |
| `completedAt` | ISO string/null | 完成时间，null 表示未完成 |
| `notes` | string | 备注（已完成事项） |
| `notesEditing` | bool | 备注是否处于编辑状态（仅前端使用） |

## 数据保护机制

### 1. 自动备份（`_backup_todos()`）

**触发时机：**
- 每次调用 `write_todos()` 保存数据前
- 服务器启动时（`ensure_data_dir()`）

**实现细节：**
- 备份文件命名：`todos_YYYYMMDD_HHMMSS.json`
- 存储位置：`DATA_DIR/backups/`
- 保留策略：最近 20 份，超出自动删除最旧的
- 空数据不备份（`len(data) == 0` 时跳过）
- 备份失败不影响正常保存（异常被捕获）

```python
def _backup_todos():
    if not os.path.exists(DATA_FILE):
        return
    data = json.load(open(DATA_FILE))
    if len(data) == 0:
        return
    os.makedirs(BACKUP_DIR, exist_ok=True)
    ts = datetime.now().strftime('%Y%m%d_%H%M%S')
    shutil.copy2(DATA_FILE, os.path.join(BACKUP_DIR, f'todos_{ts}.json'))
    # Rotate old backups
    backups = sorted([f for f in os.listdir(BACKUP_DIR) if ...])
    for old in backups[:-MAX_BACKUPS]:
        os.remove(os.path.join(BACKUP_DIR, old))
```

### 2. 原子写入（`write_todos()`）

```python
def write_todos(data):
    _backup_todos()            # 1. 先备份
    tmp = DATA_FILE + '.tmp'
    json.dump(data, open(tmp, 'w'))  # 2. 写入临时文件
    os.replace(tmp, DATA_FILE)       # 3. 原子替换
```

`os.replace()` 是原子操作，确保即使在第 2 步崩溃，原文件也不会损坏。

### 3. 空数据保护

```python
if len(data) == 0:
    existing = read_todos()
    if len(existing) > 0:
        # 拒绝保存，返回 409 Conflict
        return
```

防止浏览器旧标签页在卸载时发送空数组覆盖数据。

### 4. 数据与模板分离

- `index.html` 不包含任何嵌入数据
- `todos.json` 独立存储
- 更新 HTML 模板不会影响数据文件

## 数据恢复流程

### 自动恢复

1. 打开 `dist/To Do.app/Contents/Resources/backups/` 目录
2. 找到最新的备份文件
3. 复制覆盖上级目录的 `todos.json`
4. 刷新浏览器

### 手动恢复

1. 浏览器点击「📂 导入 JSON」
2. 选择之前导出的备份文件

### 备份文件格式

备份文件与 `todos.json` 格式完全相同，可以直接复制使用。

## 打包部署

### 打包为 .app

```bash
python3 setup.py py2app
```

输出在 `dist/To Do.app`。

### 同步更新到 .app

修改 `server.py` 或 `index.html` 后，需要同步到 `.app` 内部：

```bash
cp server.py "dist/To Do.app/Contents/Resources/server.py"
cp index.html "dist/To Do.app/Contents/Resources/index.html"
```

> 注意：同步时不覆盖 `todos.json` 和 `backups/` 目录，确保用户数据不丢失。

## 版本历史

- 空数据保护：拒绝浏览器旧标签页的空数据覆盖
